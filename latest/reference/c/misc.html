Error handling#

const char *eqs_last_error(void)#

Get the last error message that was created on the current thread.

Returns:

the last error message, as a NULL-terminated string

typedef int32_t eqs_status_t#

Status type returned by all functions in the C API.

The value 0 (EQS_SUCCESS) is used to indicate successful operations, positive values are used by this library to indicate errors, while negative values are reserved for users of this library to indicate their own errors in callbacks.

EQS_SUCCESS#

Status code used when a function succeeded

EQS_INVALID_PARAMETER_ERROR#

Status code used when a function got an invalid parameter

EQS_BUFFER_SIZE_ERROR#

Status code used when a memory buffer is too small to fit the requested data

EQS_INTERNAL_ERROR#

Status code used when there was an internal error, i.e. there is a bug inside equistore itself
